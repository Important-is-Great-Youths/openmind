## Project Rules
### Commit Message
> **[유형] : 작업 내용**

#### Type
- feat : 새로운 기능 추가
- fix : 버그 수정
- docs : 문서 관련
- style : 스타일 변경 (포매팅 수정, 들여쓰기 추가, ...)
- refactor : 코드 리팩토링
- test : 테스트 관련 코드
- build : 빌드 관련 파일 수정
- perf : 성능 개선
- chore : 그 외 자잘한 수정
### Branch Name
- BranchPropose-INI 형태
- 예시 : Mainpage-LSY
-- 목적 : 메인페이지 구현
-- 브랜치 생성자 : 이서영(LSY)
### Merge Rule
- 반드시 작업 브랜치와 main 브랜치에 모두 push 할 것
- main에 merge 할 경우 무조건 PR을 만들 것
- 팀원들은 PR을 주기적으로 확인하여 새 PR이 있으면 리뷰할 것
- 마지막으로 리뷰한 사람이 문제가 없다고 판단하면 merge할 것
